# 故障处理

核心原则：用户可以接受"还有 15 个没翻译完，原因是 XXX"，但无法接受"全部完成"后发现一半是空的。所以我们选择如实报告，而非美化结果。

## 异常日志

执行过程中，所有异常追加写入 `ai_analysis/ERRORS.md`：

```markdown
# 执行异常日志

## [阶段标记] 异常类型
- **影响范围**: 哪些提示词/文件受影响
- **原因**: 具体错误信息
- **处理**: 采取了什么措施（重试/跳过/降级）
- **状态**: 已解决 / 未解决
```

## 故障分类与处理

| 故障类型 | 识别方式 | 处理策略 | 阻断流程？ |
|---------|---------|---------|----------|
| 子代理权限拒绝 | 返回中包含 "denied"/"permission" | 重试 2 次 → 主 agent 兜底 → 记录 | 否 |
| 子代理超时 | 长时间无返回 | 拆分为更小批次重试 → 记录 | 否 |
| 子代理静默失败 | 声称完成但 Glob 找不到文件 | 重新派发 → 记录 | 否 |
| 源文件无法读取 | Read 返回错误 | 翻译文档中标注 [源文件无法读取] → 记录 | 否 |
| 写入失败 | Write 返回错误 | 重试 1 次 → 记录 | 否 |
| MANIFEST 丢失 | Glob 找不到 MANIFEST* | 重新执行第二阶段搜索 | 否，回退重做 |
| 项目无 LLM 内容 | 第一阶段未发现提示词或依赖 | 告知用户并终止 | **是** |
| 搜索结果为空 | 8 种搜索方法全部返回空 | 确认搜索模式正确后如实告知用户 | **是** |
| lark-cli 命令失败 | 返回错误码 | 检查权限/token → 重试 → 记录 | 否 |
| Mermaid 渲染失败 | whiteboard-cli 报错 | 保留原始代码块 → 记录 | 否 |
| 白板上传失败 | API 返回错误 | 重试 1 次 → 记录 → 降级为文本 | 否 |
| 节点创建失败 | API 限流或权限 | 等待 5 秒重试 → 最多 3 次 → 记录已创建的节点继续 | 否 |
| 部分节点层级已建 | 中途失败 | 用 `nodes list` 查已建节点 → 从断点继续 → 不重复创建 | 否 |
| Markdown 上传超长 | 命令行长度限制（>30KB） | 检查文件大小 → 超 30KB 自动分段 → overwrite + append | 否 |
| Markdown 被转义上传 | `docs +fetch` 中出现字面量 `\n` / `\t`，或整页变成单行 | 停止追加 → 回到原始 `.md` 文件 → 用文件内容重新 overwrite 上传 → 记录 | 否 |
| Wiki 空间无写权限 | 只有 read 权限 | 提示用户选择有权限的空间或授予写权限 | 暂停等用户 |
| git clone 目录冲突 | 目标目录已存在 | 检查是否完整克隆 → 复用/清理/改名 `<repo>_<timestamp>` | 否 |
| Windows 文件锁删除失败 | `.git/` 被进程占用 | 等 3 秒重试 → 仍失败则提示用户关闭占用进程 | 暂停等用户 |
| Mermaid 中文标点解析错误 | 全角 `（）【】` 等 | 执行语法清洗：全角→半角 + subgraph 名加引号 → 重试 | 否 |
| `/tmp` 路径不存在（Windows） | Windows 无 `/tmp` | 使用 `${TMPDIR:-/tmp}` 并 `mkdir -p` 确保存在 | 否 |
| `--scope` 授权失败 | lark-cli 不支持 --scope | 改用 `--domain wiki,docs,drive` 按域授权 | 否 |

### 子代理失败的分级重试

**第一级：重新派发子代理（优先）**
1. 检查返回状态——错误或权限拒绝时标记该批次为未完成
2. 调整 prompt 后重新派发（权限问题：拆分批次；超时：缩小单元；内容错误：补充指令）
3. 同一批次最多重试 2 次

**第二级：主 agent 兜底（仅在子代理反复失败时）**
4. 连续 2 次子代理失败后，主 agent 直接用 Write 补写
5. 兜底时只读提示词正文，跳过周围代码，避免消耗上下文

### 归档阶段（Phase 3）断点恢复

归档阶段涉及多个顺序操作（创建节点 → 上传内容 → 渲染白板），中途失败时：

1. **记录已完成的 token**：每成功创建一个节点或上传一篇文档，立即在内存中记录 token 映射
2. **从断点继续**：不需要从头创建所有节点。用 `wiki nodes list` 查询已存在的子节点，跳过已创建的
3. **白板独立重试**：Mermaid 白板渲染失败不影响文档内容——文档内容已上传，只是白板为空。可以单独重试白板上传
4. **部分成功也交付**：即使某些翻译文档未能上传，报告和说明书已上传就可以交付，在最终报告中标注哪些未完成

## 健康检查

每次关键操作后做轻量确认：
- 写文件后 → Glob 确认文件存在
- 子代理返回后 → Glob 检查文件数量 = 预期
- 根节点首页上传后 → `docs +fetch` 抽查前 20 行，确认没有字面量 `\n` / `\t`
- 阶段完成后 → 检查 ERRORS.md 有无未解决异常
- 验证通过后 → 最终文件计数 = MANIFEST 条目数

## 降级交付

经过所有重试后仍有部分无法完成时，允许降级交付，但需满足：

1. **所有失败都已记录在 ERRORS.md**
2. **最终报告包含"未完成项"清单**
3. **INDEX.md 中对未翻译的提示词标记 [未翻译]**
4. **覆盖率如实报告**（如"93%（67/72），5 个因 XXX 未完成"）

## 诚实报告原则

- 数量用精确数字（"已创建 38 个"），不用模糊词
- 子代理声称创建 N 个文件，必须 Glob 验证后才能采信
- 如果中途发现之前阶段有遗漏，回头补齐再继续
- 出现无法解决的故障，立即告知用户
